# 登場曲制御・試合記録統合アプリ（ピンポン野球サークル用）

仕様書: `登場曲制御アプリ_仕様書.md`（完全版 v2.0）に基づく Android アプリ。
v2.0 でスコアブック機能（カウント・結果記録・試合タイマー・投手成績・スプレッドシート出力）を統合した。

## ビルド方法

1. **Android Studio**（Koala 以降推奨）をインストールする
2. Android Studio で「Open」→ この `EntranceSongApp` フォルダを開く
3. Gradle Sync が走る（初回は Gradle 8.7 と依存ライブラリのダウンロードで数分かかる）
   - Gradle wrapper の jar は含めていないため、Studio が「wrapper を修復するか」と聞いてきたら許可する
4. Android 実機（Android 8.0 / API 26 以上）を USB 接続し「Run」

> **注意:** プロジェクトが OneDrive・日本語パス配下にあるとビルドが不安定になる場合がある。
> エラーが出るときは `C:\dev\EntranceSongApp` など ASCII のローカルパスで開くこと。

## 使い方（試合当日の流れ）

1. **メンバータブ**: サークル員の投打・フリガナと、打席用 mp3（1〜6曲）・投手用 mp3（最大3曲）を登録
   （初回起動時にメンバー表の31名が投打つきで自動登録される。曲とフリガナは各自で登録。
   一覧はフリガナのあいうえお順で並ぶ）
2. **設定タブ**: 球場名と、スプレッドシート連携用の Apps Script URL を設定（後述）
3. **打順タブ**: 先攻・後攻それぞれ3人以上の打順を検索して設定。試合開始前は「打順を一括削除」ボタンで両チームまとめて消去可能（試合終了後も打順は残るので、続けて同メンバーで組める）
4. **試合タブ**: 「後攻 先発投手」を選択してから「▶ 試合開始」→ **まず後攻先発投手の登場曲が流れ**、試合タイマーが0秒から開始。停止ボタンを押すか投手曲が終わると先攻1番打者の曲へ自動で切り替わる
5. **打席の記録**:
   - **ボール / ストライク / ファール** ボタンでカウントを進める（S/B/O は野球中継風の色付き球で表示）
     - 4ボール → 自動で四球確定フェーズへ（走者は自動強制進塁）
     - 3ストライク → 大きな「空振り」「見逃し」ボタンで選択
   - 打球が出たら **ヒット / アウト / エラー / nHnE / 犠牲 / 死球 / 妨害** をタップ
     （三塁走者がいて満塁でなく、0/1アウトのときのみ **スクイズ** ボタンが追加表示される）
     → **その瞬間に次打者の曲が流れ始め**、詳細入力ポップアップが出る
   - ポップアップで **打球結果（ゴロ/フライ＝進塁判定に使用）**・打球方向（0投〜6ファール）・打球性質（記録用のゴロ/ライナー/フライ）・バット種類（r,d,y,t,i,w,b,o 色分けボタン）・個別状況を入力。塁況/アウト/得点/打点は自動計算した結果を読み取り専用で表示（手動補正は廃止）
   - **確定** → 曲が2秒フェードで停止し、その瞬間が次打者の「打席開始時間」として記録される
   - 前の打者の入力を間違えたら **「◀ 前の打者」ボタン**で直前の打席をやり直せる（1手戻し）
6. **チェンジ**（ボタン or リモコン前トラック長押し1.5秒）: 攻守交代。1回裏の開始時に先攻チームの先発投手を聞かれ、入力後に先攻先発投手の登場曲→先頭打者の順で流れる
7. **投手交代**: 守備メンバー名をタップ → 投手曲（最大3曲）選択で割り込み再生+投手交代。割り込み中に前の曲（リモコン）で割り込み破棄
8. **試合終了**: 確認ダイアログ → 投手成績（失点・自責点・先発・勝敗・H・S）を自動計算し、スプレッドシートへ送信。ローカルには3試合分保持

> **画面ボタンとワイヤレスリモコンの対応（v2.1〜）**: 画面上の「前の曲/次の曲」ボタンは廃止し、「前の打者」ボタンに置き換えた。
> ワイヤレスリモコンの次トラック（手動スキップ）・前トラック（打順戻し／割り込み破棄）・長押し（チェンジ）・再生停止は従来どおり機能する。

### タイムスタンプ（動画振り返り用）

- **打席開始時間**: 前打席の確定（=曲停止）の瞬間の試合タイマー値
- **打点球投球前時間**: 得点が発生した打席のみ。結果ボタン押下時刻 − 8秒（負なら0）

### YouTube概要欄用タイムスタンプ（履歴タブ / v2.4）

履歴は**過去10試合分**保持され、各試合を開くと**YouTube概要欄にそのまま貼れるテキスト**が表示される
（「概要欄用テキストをコピー」でクリップボードへ）。

- **ハイライト**: 得点が入った打席のみ。時刻は**結果ボタン押下の10秒前**（打点を生んだ1球が映る位置）
  例: `4:50 1回裏 玉木ソロホームラン`
- **各選手の打席**: 先攻/後攻チーム別・打順ごとに、各打席の**開始時間**（前の打者の結果入力が
  終わった瞬間）を列挙。例: `1:20 第1打席 三振`
- **タイムスタンプ補正**: 動画の開始と記録の開始がずれた場合、試合ごとに −10/−1/+1/+10 秒ボタンで
  全タイムスタンプを一律にずらせる（負の値も可、0未満になる時刻は 0:00 に丸め）

## メンバー・楽曲の共有（サーバー同期 / v2.2）

複数人でアプリを使う前提で、**サークル員の名前・フリガナ・左右・登場曲を共有スプレッドシート＋共有Driveで一元管理**し、
アプリ側の設定タブ「メンバー・楽曲を同期」ボタンで一括取得する。楽曲は GAS がオーナー権限で Drive から読み出して
base64 で返すため、**Drive フォルダを一般公開しなくてよい**（オーナーがアクセスできれば OK）。

### サーバー側の準備

1. 共有スプレッドシートに **「楽曲登録」シート**を作る。1行目はヘッダー、2行目以降が1人1行:
   | A 名前 | B フリガナ | C 打 | D 投 | E〜J 打席曲1〜6 | K〜M 投手曲1〜3 |
   - 曲セルには、共有Driveにアップした mp3 の **共有リンク**（右クリック→リンクをコピー）または **ファイルID** を貼る
   - 空欄の曲スロットは無視される
2. mp3 は**サークル共有のGoogle Driveフォルダ**に置く（GASのオーナーがアクセスできる場所であればよい）
3. 下記 GAS（`doGet` に roster/song エンドポイントを含む版）をデプロイ

### アプリ側

- 設定タブの **「メンバー・楽曲を同期」** を押すと、`楽曲登録` シートを読み込み、各曲を Drive から端末内
  （`filesDir/songs/`）にダウンロードして、メンバー一覧を置き換える。以後はオフラインでも再生できる
- 楽曲は**ファイルIDごとにキャッシュ**され、2回目以降の同期では新規分だけダウンロードする
- ⚠️ 同期はメンバー一覧を**サーバーの内容で置き換える**。試合中は実行不可

## スプレッドシート連携（既存「2026春夏」形式に対応）

OAuth 設定なしで動く Google Apps Script 方式。**試合終了時に、その試合日の新しいシート**（例 `2026-06-17`、
同日の2試合目は `2026-06-17-2`）を作り、既存の試合シートと同じ **A〜AB列（1球ごと）＋ AD〜AL列（出場者・投手成績）**
の形式で書き込む。さらに **「全試合経過」** と、試合月に該当する **「◯月月間試合経過」**（例 4月なら `4-5月月間試合経過`）
へも同じ行を追記するので、既存シートの集計がそのまま働く。

### 設定手順

1. 記録用スプレッドシート（2026春夏）を開く → 拡張機能 → **Apps Script**
2. **`gas/コード.gs`（このプロジェクト内・完全版）の中身を丸ごと貼り付けて**保存
   （下に載っているスクリプトは説明用。速報サイト・ライブ送信を含む最新版は `gas/コード.gs` が正）
3. **デプロイ → 新しいデプロイ → 種類: ウェブアプリ**
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
4. 発行された WebアプリURL（`https://script.google.com/macros/s/…/exec`）をアプリの設定タブに貼る

## 野球速報サイト（v2.5）

**WebアプリURLをブラウザで開くと、そのまま速報サイトになる**（サークル員にURLを共有するだけ。
ホスティング・サーバー費用は不要）。

- **試合一覧**: 過去の試合（日付シート）と、進行中の試合（🔴試合中）を一覧表示
- **試合ページ**: イニングスコア（計/安/失）、責任投手（勝・敗・H・S）、本塁打（シーズン通算の号数つき）、
  打者成績（シーズン打率・打数・安打・打点・本塁打）、投手成績（回・球数・被安打・奪三振・四死球・失点・自責点）
- **リアルタイム速報**: アプリの設定で「リアルタイム速報送信」がONのとき、1打席確定するごとに
  LIVE シートへ送信され、試合ページが約20秒ごとに自動更新される。「前の打者」で戻すと速報側も取り消される。
  試合終了で LIVE は日付シートの正式記録に置き換わる
- 電波が無い場所では送信は静かに失敗するだけで、試合記録には影響しない（試合終了時の本保存が正）

```javascript
// 試合記録の受信用 Apps Script（2026春夏 形式）
const ALL_GAMES = "全試合経過"; // 全試合を積み上げるシート名

// A〜AB（28列）＋ AD〜AL（9列, ACは空けて33〜41列）のヘッダー
const HEADER = [
  "日付","球場","回","表裏","アウト数","塁況","得点・先","得点・後",
  "打者","打左右","投手","投左右","打席中球数","カウント","結果",
  "打球方向","打球性質","バット種類","塁打数","失策数",
  "次の回","次の表裏","次のアウト数","次の塁況","次の得点・先","次の得点・後",
  "得点","打点"
];
const PBLOCK_HEADER = ["出場者","投手名","失点","自責点","先発","勝","敗","ホールド","セーブ"];
const PBLOCK_COL = 30; // AD列

const ROSTER_SHEET = "楽曲登録"; // メンバー・楽曲の一元管理シート（タブ名）
// 楽曲登録を「成績スプシとは別ファイル」に置く場合、そのスプレッドシートIDを入れる。
// 空文字 "" にすると、GASが紐づく成績スプシ内の同名タブを読む。
const ROSTER_SS_ID = "ここに楽曲登録スプレッドシートのIDを貼る";

// ブラウザ確認用＋メンバー/楽曲の同期エンドポイント（GET）
function doGet(e) {
  const action = e && e.parameter && e.parameter.action;
  if (action === "roster") return json(getRoster());
  if (action === "song") return json(getSong(e.parameter.id));
  return ContentService.createTextOutput(
    "OK: 登場曲アプリの受信用エンドポイントです。アプリからの送信(POST)を待機中。"
  );
}

// 「楽曲登録」シートを JSON で返す
// 列: A名前 B フリガナ C打 D投 E〜J打席曲1〜6 K〜M投手曲1〜3（曲セルはDrive共有リンク or ファイルID）
function getRoster() {
  try {
    // ROSTER_SS_ID が設定されていればその別ファイルを、無ければ成績スプシを読む
    const ss = ROSTER_SS_ID ? SpreadsheetApp.openById(ROSTER_SS_ID)
                            : SpreadsheetApp.getActiveSpreadsheet();
    const sh = ss.getSheetByName(ROSTER_SHEET);
    if (!sh) return { ok: false, error: ROSTER_SHEET + " シートがありません", members: [] };
    const v = sh.getDataRange().getValues();
    const members = [];
    for (let r = 1; r < v.length; r++) {
      const name = (v[r][0] || "").toString().trim();
      if (!name) continue;
      const bat = [];
      for (let c = 4; c <= 9; c++) { const id = fileId(v[r][c]); if (id) bat.push({ id: id }); }
      const pit = [];
      for (let c = 10; c <= 12; c++) { const id = fileId(v[r][c]); if (id) pit.push({ id: id }); }
      members.push({
        name: name,
        furigana: (v[r][1] || "").toString().trim(),
        bat: (v[r][2] || "右").toString().trim(),
        throw: (v[r][3] || "右").toString().trim(),
        battingSongs: bat,
        pitchingSongs: pit
      });
    }
    return { ok: true, members: members };
  } catch (err) {
    return { ok: false, error: String(err), members: [] };
  }
}

// Drive ファイルを base64 で返す（アプリがローカル保存して再生する）
function getSong(id) {
  try {
    const f = DriveApp.getFileById(id);
    const b = f.getBlob();
    return { ok: true, name: f.getName(), dataBase64: Utilities.base64Encode(b.getBytes()) };
  } catch (err) {
    return { ok: false, error: String(err) };
  }
}

// 共有リンク or 生ID から Drive ファイルIDを取り出す
function fileId(v) {
  if (!v) return "";
  const m = v.toString().match(/[-\w]{25,}/);
  return m ? m[0] : "";
}

function doPost(e) {
  try {
    const d = JSON.parse(e.postData.contents);
    // 拡張機能→Apps Script で作った場合はこれで開ける。
    // もしスタンドアロンのスクリプトなら SHEET_ID を入れて openById を使う（下記コメント参照）
    const ss = SpreadsheetApp.getActiveSpreadsheet();
    // const ss = SpreadsheetApp.openById("ここにスプレッドシートのID");
    if (!ss) throw new Error("スプレッドシートを取得できません（スタンドアロン script なら openById を使ってください）");

    // 1) 試合日ごとの新シート（重複したら -2, -3 …）
    let name = d.date, i = 2;
    while (ss.getSheetByName(name)) { name = d.date + "-" + i; i++; }
    const sheet = ss.insertSheet(name);
    sheet.getRange(1, 1, 1, HEADER.length).setValues([HEADER]);
    sheet.getRange(1, PBLOCK_COL, 1, PBLOCK_HEADER.length).setValues([PBLOCK_HEADER]);
    writeGame(sheet, 2, d);

    // 2) 全試合経過へ追記
    const allSheet = ss.getSheetByName(ALL_GAMES);
    appendGame(allSheet, d);

    // 3) 該当月の「◯月月間試合経過」へ追記（"4-5月月間試合経過" なら 4〜5月が対象）
    const month = parseInt(d.date.split("-")[1], 10);
    const monthly = findMonthlySheet(ss, month);
    if (monthly) appendGame(monthly, d);

    SpreadsheetApp.flush();
    return json({ ok: true, sheet: name, allGames: !!allSheet, monthly: monthly ? monthly.getName() : null });
  } catch (err) {
    return json({ ok: false, error: String(err) });
  }
}

// JSON を返す共通ヘルパー
function json(o) {
  return ContentService.createTextOutput(JSON.stringify(o)).setMimeType(ContentService.MimeType.JSON);
}

// 指定シートの末尾に、1試合分（A〜AB と AD〜AL）を書き込む
function appendGame(sheet, d) {
  if (!sheet) return;
  const start = sheet.getLastRow() + 1;
  writeGame(sheet, start, d);
}

// "1" などの数値文字列は数値へ（集計の SUM/SUMIF を効かせるため）。"3-2" や "1塁打" は文字列のまま
function num(v) {
  return (v !== "" && /^-?\d+(\.\d+)?$/.test(v)) ? Number(v) : v;
}

// startRow から A〜AB の打席行と、AD〜AL の投手/出場者ブロックを書き込む
function writeGame(sheet, startRow, d) {
  if (d.playRows.length > 0) {
    const rows = d.playRows.map(r => r.map(num));
    sheet.getRange(startRow, 1, rows.length, HEADER.length).setValues(rows);
  }
  const n = Math.max(
    d.roster.length, d.pitchers.length, d.starters.length,
    d.holds.length, d.saves.length, 1
  );
  const block = [];
  for (let r = 0; r < n; r++) {
    block.push([
      d.roster[r] || "",
      d.pitchers[r] ? d.pitchers[r].name : "",
      d.pitchers[r] ? d.pitchers[r].runs : "",
      d.pitchers[r] ? d.pitchers[r].earned : "",
      d.starters[r] || "",
      r === 0 ? (d.win || "") : "",
      r === 0 ? (d.loss || "") : "",
      d.holds[r] || "",
      d.saves[r] || ""
    ]);
  }
  sheet.getRange(startRow, PBLOCK_COL, n, PBLOCK_HEADER.length).setValues(block);
}

// "4-5月月間試合経過" / "6月月間試合経過" などから、month を含むシートを探す
function findMonthlySheet(ss, month) {
  const re = /^(\d+)(?:-(\d+))?月月間試合経過$/;
  const sheets = ss.getSheets();
  for (let i = 0; i < sheets.length; i++) {
    const m = sheets[i].getName().match(re);
    if (!m) continue;
    const lo = parseInt(m[1], 10);
    const hi = m[2] ? parseInt(m[2], 10) : lo;
    if (month >= lo && month <= hi) return sheets[i];
  }
  return null;
}
```

- 送信は試合終了時に自動実行。圏外などで失敗した場合は**履歴タブ → 該当試合 → 再送信**
- 月間シートが見つからない場合は日付シートと全試合経過のみ更新する（月間シートは手動でコピー）

### 出力される列・語彙（既存シートに一致）

- **塁況（F・X列）**: `無塁` / `一塁` / `二塁` / `三塁` / `一二塁` / `一三塁` / `二三塁` / `満塁`
- **カウント（N列）**: 決め球の前のカウント `B-S`（例: フルカウントから四球 = `3-2`）
- **打球方向（P列）**: 0:投 1:左 2:左中 3:中 4:右中 5:右 6:ファール
- **結果（O列）**: `1塁打`〜`4塁打`（安打／本塁打=4塁打） / `ゴロ`・`フライ`・`ライナー`（アウト） /
  `併殺` / `失策` / `犠飛` / `スクイズ` / `死球` / `四球` / `空三振`・`見三振`
- **塁打数（S列）**: 安打はその塁打数、nHnE は安打分のみ、それ以外は0。**失策数（T列）** は失策での進塁数
- **投手成績（AI〜AL列）**: 勝・敗・ホールド・セーブは「その投手の名前」を記入

## 進塁・成績計算の前提（要確認事項）

「走者は基本的に打者走者と同数進塁する」ルールに基づくベースライン実装。
自動計算が実態と違う場合も、**確定前のポップアップで塁況・アウト・得点・打点を手で補正できる**。

- ヒット/エラー/nHnE: 全走者が塁打数（nHnE は H+E）と同数進塁
- 四球/死球/妨害: 押し出し（詰まっている走者のみ1つ進む）
- 進塁が起こるかどうかは、打球性質（記録用）ではなく **打球結果（ゴロ/フライ）** で判定する。
  打球性質がフライでも1バウンドさせて捕った場合は打球結果に「ゴロ」を選ぶと進塁が起こる
- **ゴロ（打球結果）アウト**: 走者は1つ進塁。ただし三塁走者は「ワンファンブル」選択時のみ生還、
  それ以外は三塁に留まる（塁が空かない後続走者も止まる）
- **フライ（打球結果）アウト + タッチアップ**: 真ん中より右か左かの2択（審判判断）。
  ライト側は三塁走者が生還し二塁走者は三塁へ、レフト側は三塁走者のみ生還。一塁走者はそのまま。
  打球方向が右中・右なら自動でライト側、左・左中なら自動でレフト側、
  「中」を選んだ場合のみポップアップに右側/左側の選択ボタンが出る（選択必須）
- ライナーアウト: 走者そのまま
- **併殺**: アウト+2（先頭走者+打者走者）。1塁/1・2塁/満塁は先頭走者を除去し残走者は1つ進塁。
  **1・3塁のみ選択制**: 「併殺」を選ぶと一塁走者+打者アウトで三塁走者は生還、
  「三塁走者アウト」を選ぶと三塁走者を封殺して打者走者は一塁に生きる（アウト+1）
- **犠牲（送りバント・犠飛）**: アウト+1、走者は1つ進塁。三塁走者は生還しない
  （三塁走者を還すプレーは「スクイズ」で記録）。フライ+タッチアップ選択時は上記の左右ルール
- **スクイズ**: 三塁走者がいて満塁でないときのみ結果ボタンに表示。ポップアップで成功/失敗を選択（必須）
  - 成功: 打者アウト（+1）、三塁走者が生還し他の走者も1つ進塁
  - 失敗: 三塁走者が本塁でアウト（+1）、打者走者は一塁に生きて他の走者は押し出し
- **2アウト・フルカウント（一塁に走者あり）**: 走者自動スタート。安打系（ヒット/エラー/nHnE）では
  **フォースされた走者のみ**が塁打数より+1余分に進塁する（塁より手前が全て埋まっている走者）。
  例: 一三塁なら一塁走者だけ+1、三塁走者は通常進塁。二塁/三塁/二三塁/無塁は対象外（ポップアップに⚡表示）
- 打点: 発生得点と同数（エラーの得点は打点0）。手動補正可
- **失点**: 走者を出塁させた投手の責任（継投後に生還しても前の投手に付く。公式規則準拠）
- **自責点**: 公式規則準拠のイニング再構成方式。①失策・打撃妨害で出塁した走者の生還、
  ②失策が無ければ3アウトでイニングが終わっていた後の得点、③失策による進塁が無ければ
  本塁に届かなかった走者（nHnEの安打部分だけでは生還できない走者）の得点 — を自責から除外
  （※守備側のワンファンブルは失策に数えない運用のため自責に含む）
- **勝**: 勝利チームが最後に勝ち越した時点の登板投手（その時点で未登板なら先発）
- **敗**: 決勝点の責任投手（その走者を出塁させた投手）
- **セーブ**: 勝利チームの最終投手（勝利投手・先発を除く）で、1アウト以上を記録し、登板時に
  (a)3点差以内のリード (b)同点の走者が塁上・打者・次打者にいる状況のリード (c)3イニング以上投球
  のいずれかを満たす場合（公式規則準拠）
- **ホールド**: 勝利チームの救援（勝敗・セーブ投手を除く）で、上のセーブ要件と同じ状況の
  リード時に登板して1アウト以上を記録し、降板までリードを守り切った場合

## 実装メモ

- **データ永続化**: `filesDir` 配下の JSON（members / state / history / settings）。アプリを落としても試合状態・タイマー・入力途中の結果ポップアップまで復元される
- **試合タイマー**: 開始時刻の epoch を保存し経過秒を都度計算するため、バックグラウンドでも狂わない
- **リモコン対応**: フォアグラウンドサービスが MediaSession を保持。詳細入力ポップアップ表示中はリモコンの次/前/チェンジ操作を無視する（再生停止のみ有効）
- **3アウト目の自動チェンジ**: アウト/三振で3アウト目が確定した場合は、結果ボタン押下の時点で相手チーム先頭打者の曲が流れ、確定と同時にチェンジ処理が走る。併殺入力で後から3アウトになった場合は手動でチェンジを押す（直前に進めた打順は自動で巻き戻る）
- **mp3の権限**: SAF で選択し永続 URI 権限を取得。ファイルを移動・削除すると再生不可
- **OCR**: ML Kit 日本語テキスト認識（オンデバイス）

## 既知の割り切り（v2.0）

- 通知アイコンは Android 標準アイコンを流用
- 打順の並べ替えはドラッグ非対応（削除→挿入で対応)
- 盗塁・暴投など打席の間の走者移動は記録対象外（次打席の確定時に塁況を補正して吸収する）
- 失点・自責点・勝敗・H・S はプロ野球の公式規則準拠（上記参照）。走者の責任投手はイニングの
  塁況遷移から再構成しているため、盗塁・暴投等の記録外の走者移動があると帰属が近似になる
